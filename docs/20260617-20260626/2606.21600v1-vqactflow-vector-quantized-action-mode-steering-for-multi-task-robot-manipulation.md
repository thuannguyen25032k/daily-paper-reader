## Abstract
Multi-task robot manipulation policies are challenging to learn from demonstration because traditionally a single network must select among qualitatively different action modes from a multimodal demonstration distribution, conditioned on language and visual context. A wrong mode selection means executing the wrong task or an action infeasible in the scene. Tokenizing continuous actions into a learned discrete codebook separates these modes at the representation level, offering structural advantages for multi-task learning. We propose VQActFlow, a multi-task manipulation policy that tokenizes action chunks and generates code sequences via Variational Flow Matching. VQActFlow maintains an explicit preference over action modes throughout generation. Inference-time guidance acts on this preference to steer mode commitment. We instantiate this with classifier-free guidance over language conditioning, which steers the policy toward the instructed action mode, and a learned codebook critic that supplies a complementary feasibility signal. We evaluate VQActFlow on three platforms: the LIBERO simulation benchmarks, a Unitree G1 humanoid performing whole-body pick-and-place, and an ALOHA-style bimanual platform performing contact-rich tasks. Across these benchmarks, VQActFlow outperforms both continuous and discrete baselines.

## 2. Methodology

- **Core idea**: Use a frozen VQ-VAE to tokenize action chunks into discrete codebook indices, then train a Variational Flow Matching (VFM) policy to generate code sequences. The VFM policy produces logits over codebook entries at every integration step, giving an explicit categorical preference over action modes. Inference-time guidance (CFG for language conditioning and a codebook critic for feasibility) steers this preference toward the correct mode.

- **Key technical details**:
  - VQ-VAE: Encoder `E` maps action chunk `a_{1:H}` to continuous latents `z ∈ R^{L×d_e}` (L = H/2^S after S temporal downsampling stages). Each latent is quantized to its nearest codebook entry from a codebook of size `K=512` (embedding dimension `d_e`). Decoder `D` reconstructs the action chunk from the quantized embeddings.
  - VFM Policy: During training, given target embeddings `x_1 = z_q` and ground-truth indices `{k_l^*}` from the frozen encoder, a linear interpolant `x_t = (1-t)x_0 + t x_1` is constructed (`x_0 ~ N(0,I)`). A Diffusion Transformer (DiT) backbone `f_θ` takes `(x_t, t, observation, language)` and outputs logits `h ∈ R^{L×K}`. Training loss is cross-entropy over code indices:  
    `L_VFM = -Σ_l [h^{(l)}_{k_l^*} - log Σ_j exp(h^{(l)}_j)]`.  
    During inference, logits are converted to probabilities `p^{(l)}_k`, and the posterior mean is computed: `μ_1^{(l)} = Σ_k p^{(l)}_k e_k`. The velocity field is `v_θ = (μ_1 - x_t)/(1-t)`. Euler integration from `t=0` to `t=1` produces `x_1`, which is quantized to nearest code, then decoded to actions.
  - Guidance:  
    - **CFG**: On logits: `̃h = h_∅ + w·(h_c - h_∅)` (w≥1), then guided probabilities are used in Eq. 7.  
    - **Codebook Critic**: A small transformer encoder `C_ψ(o,c,p,t)` outputs a scalar feasibility score. Gradient of this score w.r.t. `p` is added to the probability vector: `̃p = Proj_Δ [p + λ(t)·Π_T ∇_p C_ψ]` where `λ(t) = λ_max (1 - H(p)/log K)` is entropy-adaptive. Training uses InfoNCE loss with multiple negative types (temporal shuffle, random code replacement, wrong observation, cross-demo swap, in-episode shift, wrong language, wrong vision).

## 6. Main Conclusions and Findings

- VQActFlow with CFG at moderate weight (w=4) achieves 81% on LIBERO-Goal, significantly outperforming Discrete Policy (61.5%).
- On LIBERO-90, combined CFG (w=2) + critic (λ=1) reaches 80.5%, surpassing continuous and discrete baselines.
- On Unitree G1, CFG raises success from 23.8% to 57.5%, primarily by reducing wrong-task errors (41.3% → 2.5%).
- On bimanual platform, VQActFlow (w=4 + λ=1) achieves 77.5% vs. Discrete Policy 61.3%. Continuous CFM fails due to high-frequency oscillations (peak jerk ~10^4 rad/s^3 vs. VQActFlow ~2×10^2 rad/s^3).
- The codebook critic provides complementary improvement (~+3% on LIBERO-90, +15% on bimanual) over CFG alone.
- The VQ bottleneck and categorical interface are key to both mode separation and motion smoothness.

## 7. Strengths

- **Novel methodology**: Combines VQ-VAE tokenization with Variational Flow Matching to maintain explicit categorical preference during generation, enabling targeted inference-time guidance.
- **Two guidance mechanisms**: CFG for task selection and codebook critic for feasibility, both acting on the same discrete preference interface; shown to be complementary.
- **Extensive evaluation**: Tested on two simulation benchmarks and two distinct real hardware platforms (humanoid and bimanual) with contact-rich tasks.
- **Practical inference speed**: Full guidance latency (350 ms) well within action chunk span (1.28 s), allowing asynchronous execution.
- **Motion smoothness**: VQActFlow generates significantly smoother trajectories than continuous flow matching, critical for hardware deployment.

## 8. Limitations

- **VQ-VAE bottleneck**: Information loss from discretization ultimately bounds policy performance; may limit applicability to very fine-grained or high-frequency actions.
- **Codebook critic negative design**: The set of negative types (temporal shuffle, random code replacement, etc.) is manually chosen; other missing violation dimensions could affect generalizability.
- **Compute/resources not reported**: Lack of transparency on training cost; prevents replication and resource estimation.
- **Limited hardware task count**: Only 4 tasks per platform; scaling to more diverse or long-horizon tasks is not demonstrated.
- **Data source**: Relies on VR teleoperation data; no analysis of data quality or domain shift.
- **CFM baseline failure**: Not directly compared on bimanual because it failed to complete any task; the comparison is only against Discrete Policy, not against other modern continuous policies (e.g., Diffusion Policy) on hardware.
