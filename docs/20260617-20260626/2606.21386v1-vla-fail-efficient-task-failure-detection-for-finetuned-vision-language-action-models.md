## Abstract
Vision-language-action models (VLAs) achieve state-of-the-art performance on many robotic manipulation tasks, yet they can still behave unpredictably in out-of-distribution scenarios. Runtime failure detection is therefore essential for the safe real-world deployment of VLAs. However, existing task failure detectors require computationally expensive action sampling, are based on architectural assumptions that limit their applicability to VLAs, or need access to failure rollouts. We propose VLA-FAIL, a lightweight and broadly applicable failure detection framework for VLAs that combines two novel failure detectors with minimal overhead, without requiring failure data. The first, last-layer Mahalanobis distance (LLMD), detects out-of-distribution states by measuring token-wise deviations in last-layer features relative to the training data. The second, action chunk consistency (ACC), exploits the temporal overlap induced by receding-horizon control and detects failures when consecutive action chunks become inconsistent. To capture the trade-off between detection accuracy and detection latency, we introduce AUCPDT, a threshold-independent metric that jointly evaluates precision, recall, and detection time. Through extensive real-world and simulation experiments, we demonstrate that LLMD and ACC capture complementary failure modes whose combination enables reliable and early failure detection across diverse tasks, frequently outperforming significantly more expensive baseline methods.

### 1. Core Problem and Overall Meaning
Vision‑language‑action models (VLAs) can fail unpredictably when encountering out‑of‑distribution (OOD) scenarios. Existing runtime failure detectors either require expensive multiple action samples, are tied to specific architectures, or need costly failure rollouts for training. **VLA‑FAIL** addresses these issues by providing a lightweight, broadly applicable failure detection framework that incurs minimal computational overhead and does **not** require any failure data. Its main goal is to enable safe real‑world deployment of VLAs by detecting task failures early and reliably during execution.

### 2. Methodology
- **Core idea** – Combine two complementary signals:  
  1. **Last‑Layer Mahalanobis Distance (LLMD)** – Measures token‑wise deviations of last‑layer features from the training‑data feature distribution.  
  2. **Action Chunk Consistency (ACC)** – Quantifies velocity‑normalized disagreement between overlapping parts of consecutive action chunks under receding‑horizon control.  

- **Key technical details**  
  - **LLMD**:  
    - Uses a **fixed prior noise** sample to isolate observation‑dependent features from flow‑matching noise.  
    - Computes token‑wise mean (`μ_s`) and regularized covariance (`Σ_s`) from training data.  
    - Score at time `t`: `s_t^LLMD = max_s [ (f^*(o_t)_s - μ_s)^T (Σ_s^λ)^(-1) (f^*(o_t)_s - μ_s) ]`.  
  - **ACC**:  
    - Computes absolute error between overlapped actions from successive chunks, normalized by per‑dimension average velocity (clamped to `v_min`).  
    - Smoothed with exponential moving average (α=0.9) to avoid spurious triggers.  
  - **FAIL**:  
    - Alert occurs if **either** `s_t^ACC ≥ τ_ACC` **or** `s_t^LLMD ≥ τ_LLMD`.  
    - Thresholds are set via a constant conformal prediction band on calibration data.  

- **No auxiliary model or failure data required** – only a single forward pass per timestep (shared with action generation).

- **Evaluation metrics**:  
  - **AUCPR** – area under precision‑recall curve (threshold‑independent).  
  - **AUCPDT** – newly introduced metric that jointly penalizes late detection and low precision.

### 6. Main Conclusions and Findings
- **FAIL (LLMD + ACC) achieves reliable early detection** across tasks, frequently outperforming significantly more expensive baselines.  
- LLMD and ACC capture **complementary failure modes**:  
  - LLMD detects OOD states early even when actions appear smooth.  
  - ACC detects erratic replanning but may trigger later (requires smoothing).  
- **AUCPDT** reveals that detection time matters – methods with similar AUCPR can have very different detection latencies.  
- The framework is **lightweight and practical** for real‑time deployment, requiring no failure data or external models.

### 7. Strengths
- **Computationally efficient** – only a single forward pass per timestep, sharing computation with action generation.  
- **No failure data needed** – only a preprocessing pass over successful finetuning data.  
- **Architecture‑agnostic** – applicable to flow‑matching, diffusion, and discrete VLAs.  
- **Novel metric (AUCPDT)** – captures the precision‑latency trade‑off in a threshold‑independent way.  
- **Thorough empirical validation** – real‑world + large‑scale simulation, two VLAs, multiple baselines, and extensive ablations.  
- **Real‑time ready** – in contrast to baselines that require multiple action samples.

### 8. Limitations
- **LLMD** requires access to finetuning data for computing feature statistics; may be infeasible if data is unavailable or very large.  
- **ACC** relies on receding‑horizon control with sufficient action overlap; not applicable to fully open‑loop chunk execution.  
- Can miss failures that are both **feature‑consistent and action‑consistent** (e.g., robot confidently ignoring a language instruction).  
- **No semantic reasoning** – unlike VLM‑based methods, VLA‑FAIL cannot detect semantic‑level task failures.  
- The threshold calibration uses a **constant conformal band**, which may not be optimal for episodes of varying length (though authors argue it is necessary for tasks like Drawer).
