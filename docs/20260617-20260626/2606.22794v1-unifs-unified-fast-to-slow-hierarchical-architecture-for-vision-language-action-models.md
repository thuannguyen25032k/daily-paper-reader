## Abstract
Mainstream Fast-Slow dual system vision-language-action models decouple a high-frequency action expert from a low-frequency vision-language model for efficiency, yet they face a fundamental frequency dilemma: large update gaps cause semantic drift from stale context, while small gaps erode the intended computational savings. Moreover, because the action expert receives only the VLM's final-layer representation at a single fixed frequency, rich intermediate features are discarded, limiting both information coupling and manipulation precision. Inspired by multi-timescale neural processing in the human brain, we introduce UniFS, a unified fast-to-slow architecture that resolves these challenges through three key designs. First, we stratify the VLM layers into groups with progressively decreasing update frequencies, enabling shallow layers to capture fast-changing dynamics while deeper layers cache stable semantic context. Second, a latent vector inversion mechanism re-routes the interaction order between multi-scale VLM features and the action expert, aligning fast-varying representations with fine-grained action decoding and slow-varying ones with coarse planning. Third, a multi-level supervision strategy enforces a coarse-to-fine learning hierarchy across temporal scales. Together, these designs enable richer cross-frequency information transfer within a single backbone, while the low-frequency pathways additionally preserve temporal context across steps. Experiments on LIBERO show that UniFS achieves state-of-the-art performance (98.3\% average success rate, a 2.5\% gain over VLA-Adapter baseline) while reducing average inference latency from 36.5~ms to 17.8~ms (2.1$\times$ speedup). Real-robot experiments on a Franka platform further validate its practical applicability. Code is opensourced at https://github.com/linsun449/UniFS.

## Core Problem and Overall Meaning
- **Motivation:** Current Fast-Slow dual-system VLA models (e.g., GR00T-N1, Fast-in-Slow) decouple a high-frequency action expert from a low-frequency VLM to reduce latency, but they suffer from a *frequency dilemma*: large update gaps cause semantic drift due to stale context, while small gaps erase the computational savings. Additionally, the action expert receives only the VLM's final-layer representation, discarding rich intermediate multi-scale features and limiting information coupling.
- **Goal:** Design a unified architecture that naturally produces multi-timescale latent representations from a single VLM backbone, enabling efficient inference without sacrificing performance, and providing implicit memory.

## Methodology: Core Idea and Key Technical Details
- **Core idea:** Inspired by multi-timescale neural oscillations in the human brain (e.g., gamma/beta waves), UniFS stratifies VLM layers into groups with *progressively decreasing update frequencies*. This allows shallow layers to capture fast-changing sensory dynamics and deeper layers to cache stable semantic context.
- **Three key designs:**
    1.  **Fast-to-Slow Architecture (FSA):** Layer groups are assigned update intervals $n_1 < n_2 < \dots < n_K$ (e.g., 1,2,4,8,16 timesteps). A layer group is recomputed only when the timestep is divisible by its interval; otherwise its output is reused. This yields multi-timescale latent vectors $V_{f2s} = \{h^{(t)}_k\}$.
    2.  **Latent Vector Inversion (LVI):** Empirical observation shows deeper layers naturally have *higher* temporal frequency (due to proximity to fast action output). To make deeper layers cacheable (slow), LVI inverts the interaction order: initial noisy action proposals interact with deeper features (slow), and refined actions interact with shallower features (fast). This aligns fast-varying representations with fine-grained decoding and slow ones with coarse planning.
    3.  **Multi-Level Supervision (MLS):** A loss is applied at multiple frequency groups inside the action expert (not only the final output). This enforces a coarse-to-fine learning hierarchy, preventing shortcuts through fast components.
- **Training strategy:** *Temporal Batch Sampling* randomly selects K timesteps from a contiguous window in a trajectory, preserving causality. *Frequency Feature Replacement* (FFR) performs a fully parallel forward pass for all timesteps, then retrospectively aligns features to their prescribed update frequency via indexing, decoupling computation parallelism from update asynchrony.
- **Inference:** Latent vectors from low-frequency groups are cached and reused; only final layers requiring high-frequency updates are executed every timestep, achieving ~2.6× theoretical speedup.

## Main Conclusions and Findings
- UniFS achieves **state-of-the-art 98.3% average success rate** on LIBERO, a **2.5% absolute gain** over the VLA-Adapter baseline.
- Inference latency reduces from 36.5ms to **17.8ms** (2.1× speedup), with theoretical speedup ~2.6×.

## Strengths
- **Bio-inspired design:** Directly addresses the frequency dilemma by smoothly varying update frequencies across layers, avoiding rigid module separation.
- **Unified single backbone:** Eliminates feature alignment issues between separate VLM and action expert, enabling better information coupling.
- **Implicit memory:** Low-frequency pathways naturally preserve historical context without explicit memory buffers, enabling performance on memory-requiring tasks (e.g., Exchange Boxes).
- **Robust to configuration changes:** Frequency ratios can differ from training without major performance drop (up to 24×).
- **Open-sourced code** for reproducibility.

## Limitations
- **No training speedup:** Frequency Feature Replacement (FFR) requires full parallel forward passes, so training remains as costly as standard VLA models.
- **Large batch size requirement:** Multi-frequency learning demands sufficient temporal diversity within each batch (global batch 64).
- **Potential distribution shift:** Inverted latent vectors may interfere with pre-trained VLM semantic knowledge, requiring careful fine-tuning.
- **Limited real-robot evaluation:** Only three tasks on Franka, with modest success rates (50–75%). Generalization to more complex dynamic environments is not demonstrated.
- **Single backbone choice:** Only Qwen2.5-0.5B is used; extension to larger backbones (e.g., π0) is left for future work.
