## Abstract
A key bottleneck in training generalist policies for bimanual dexterous manipulation is the lack of large-scale, high-quality datasets. Synthetic data generation in simulation provides a scalable alternative to human video demonstrations by overcoming challenges such as morphology mismatch, missing physical interactions, and the generation of robot actions. However, existing approaches based on human teleoperation offer limited task diversity, as object-centric trajectory matching often neglects the feasibility of robot execution. Reinforcement learning (RL) enables broader scalability but is often constrained by handcrafted, task-specific rewards. In this work, we propose a systematic RL-based data generation pipeline that integrates generalizable reward design, effective domain randomization, and language-conditioned task annotations. This pipeline synthesizes diverse, high-quality datasets for dexterous bimanual manipulation and enables training of language-conditioned multi-task policies. Our experiments show that the generated data significantly improves generalization across three representative manipulation tasks.

## Methodology

### Overall Framework (RLDC – Reinforcement Learning as Data Collector)

- **Teacher-Student Paradigm**:
  1. Train per-task state-based RL teachers (Independent PPO).
  2. Roll out teachers with extensive domain randomization to collect diverse trajectories.
  3. Automatically generate language labels and augment them with a large language model (LLM).
  4. Distill the multimodal dataset into a single multi-task diffusion student policy.

### Generalizable Reward Design (Teacher Stage)

Four reusable reward terms, applied with the same scaling factors across all tasks:

- **Initial Manipulation Hand Pose**: Encourages alignment of the robot hand to a predefined pose attached to the object:
  \[
  r_{\text{align}} = -\alpha \|p_{\text{palm}} - p_{\text{def}}\|^2 - \beta d_{\text{quat}}(q_{\text{palm}}, q_{\text{def}})
  \]
- **Object Contact**: Minimizes distance between fingertips and object center of mass:
  \[
  r_{\text{contact}} = \sum_{i=1}^n \frac{1}{1 + \alpha_i \|p_{\text{tip}}^i - p_{\text{com}}\|^2}
  \]
- **Object Goal Poses with Lifting Gate**: Rewards progress toward a flexible goal pose only after the object is lifted above a height threshold \(\tau\):
  \[
  r_{\text{goal}} = \begin{cases}
  \max(d_z, 0), & \text{if } d_z < \tau \\
  \exp(-\alpha \|p_o - p_g\|^2 - \beta d_{\text{quat}}(q_o, q_g)), & \text{otherwise}
  \end{cases}
  \]
- **Robot Reset**: Encourages returning to a predefined configuration after task completion:
  \[
  r_{\text{reset}} = \exp(-\alpha \|\theta - \theta_{\text{reset}}\|^2)
  \]

### Data Collection with Domain Randomization

- **Physical randomization**: Object pose, friction, mass, restitution.
- **Visual randomization**: Color, reflectivity, roughness, camera viewpoints, lighting.
- **Objects**: 100 different complex geometries (cars, shoes, toys) scaled to similar graspable sizes, 20 color categories.
- **Language labeling**: Template-based instructions ("Use right arm to pick A, left arm to pick B") further diversified using GPT-based paraphrasing.

### Multi-Task Policy Distillation

- **Input**: 6-channel point cloud (RGB-D, 2048 points via random downsampling), robot state (joint positions/velocities), language tokens (SigLip-encoded).
- **Architecture**:
  - PointNet++ encoder (without max pooling) → spatial features.
  - Transformer with decoupled action queries for each arm (linear cross-attention complexity).
  - Separate diffusion heads for left and right actions.
  - DDIM inference (20 training steps, 5 inference steps).
- **Training loss**:
  - Action diffusion noise prediction loss.
  - Auxiliary state prediction loss: \(\ell_{\text{aux}} = \|\hat{x}_{\text{obj}} - x_{\text{obj}}\|^2\) (6D rotation representation).
- **Bimanual control**: Decoupled action queries allow independent yet coordinated arm movements.

## Experiments

### Tasks and Setup

- **Three representative tasks**: Box lifting (single-object, coordination), Multi-object pick-and-place (clutter, detection), Drawer insertion (articulated object, long-horizon).
- **Robot**: Two 6-DOF xArm UF850 arms, each with 16-DOF Allegro Hand V4 (total 44 DOF). PD control at 120 Hz, policy inference at 20 Hz.
- **Perception**: Single egocentric ZED2i RGB-D camera.
- **Simulator**: NVIDIA IsaacLab with a digital twin of the real setup.

### Baselines Compared

- **Reward design baselines** (teacher stage):
  - Task-specific rewards (finely tuned per task).
  - Object reference trajectory reward (VividEx [27]).
- **Policy baselines** (student stage):
  - DP3 [42]: 3D diffusion policy with linear point cloud encoder.
  - iDP3 [43]: with pyramid convolutional encoder.
- **Ablations**: Separate vs. shared action heads; with/without auxiliary state prediction loss; dataset size (500, 2000, 5000 trajectories).

## Main Conclusions and Findings

1. **Generalizable rewards** can achieve performance comparable to finely tuned task-specific rewards, significantly reducing engineering overhead.
2. **RL-based data generation** with extensive randomization yields diverse, robot-feasible trajectories that support zero-shot sim-to-real transfer.
3. **Multi-task student policy** trained solely on synthetic data can follow diverse language instructions and generalize to novel (out-of-distribution) objects.
4. **Decoupled action heads** improve real-world robustness by isolating failures per arm.
5. **Auxiliary state prediction loss** provides a unique advantage of simulation (ground-truth states) to anchor visual representations.
6. **Dataset scale matters**: larger datasets (5000 trajectories) substantially improve performance, especially for complex tasks.
