## Abstract
We present ZeroDex, a zero-shot framework for long-horizon dexterous manipulation that grounds language instructions into executable 3D task plans from calibrated multi-view RGB images. Rather than training an end-to-end policy, our system uses a vision-language model (VLM) to produce reference-frame task grounding and primitive-level 2D keypoints, then lifts them into 3D via multi-view fusion. This lifting combines triangulation of view-wise VLM groundings with reference-view ray voting, which searches along a semantic camera ray for geometrically consistent candidates across neighboring views. The resulting 3D keypoints support both pick-and-place and tool-use: for tool-use, we retrieve an object-centric atomic action corresponding to the inferred skill category and align its stored 6D tool trajectory to the scene; for dexterous execution, we expand the lifted grasp keypoint into a task-conditioned grasp affordance region and generate feasible grasp-motion pairs with an arm-hand motion generator. Real-world experiments show improved 3D grounding accuracy and execution reliability over single-view RGB-D grounding and fine-tuned VLA baselines. We further demonstrate long-horizon manipulation through closed-loop status verification and replan, enabling zero-shot execution on unseen objects and tool-use tasks in novel scenes.

### Core Problem and Motivation
- **Problem**: Long-horizon dexterous manipulation from high-level language instructions requires accurate 3D grounding (contact points, tool trajectories, placement targets). Existing methods (end-to-end VLA models) need extensive training data; human demonstration retargeting suffers from embodiment gap.
- **Goal**: Achieve **zero-shot** generalization to unseen objects and tasks without task-specific training or fine-tuning, using only calibrated multi-view RGB images and a pre-trained VLM.

### Methodology
- **Core Idea**: Decouple semantic reasoning (VLM) from physical execution (motion primitives). The VLM produces a task plan with 2D keypoints; these are lifted to 3D via multi-view fusion.
- **Key Technical Details**:
    1. **Reference-Frame Grounding**: VLM selects a reference view, infers manipulation mode (pick‑and‑place or tool‑use), and predicts a grounding tuple (target object, destination pixel, skill category).
    2. **Multi-View 3D Lifting**:
        - For each 2D keypoint, query VLM across all views to get view‑wise predictions.
        - **RANSAC‑style triangulation**: Pairwise candidate points scored by reprojection inlier count; accept if consensus exceeds threshold τ_tri.
        - **Reference‑view ray voting**: Sample depth candidates along the reference camera ray; VLM selects best‑matching ones from other views; aggregate votes for robust estimate.
        - Final 3D point = triangulated candidate if consensus high, else fallback to voting estimate.
    3. **Atomic Action Alignment** (tool‑use): Retrieve a stored 6D tool trajectory from a **Bag of Atomic Actions** (indexed by skill category like “pouring”). Align its start/end anchor points to scene keypoints via a rigid transformation.
    4. **Dexterous Grasp Generation**: Project grasp keypoint to all views, query VLM for 2D affordance bounding boxes; lift to 3D by multi‑view voting on mesh vertices; filter vertices with inclusion score ≥ τ. Then generate candidate grasps (cylindrical template for handles; BODex optimizer with affordance loss) and filter by kinematic/collision feasibility.
- **Formulas/Algorithm Flow** (text only):
    - Triangulation score: \( S_{tri}(a,b) = \sum_v \mathbf{1}[ \|\pi_v(X_{a,b}) - p_v\| \leq \epsilon_{tri} ] \)

    - Collision‑aware refinement: \( X^*_{loc} = \arg\min_{X'} \|X' - X_{loc}\|_2 \) s.t. penetration depth = 0.

### Main Conclusions
- ZeroDex achieves zero‑shot dexterous manipulation across diverse tasks, including tool‑use, with long‑horizon closed‑loop execution.
- Multi‑view fusion significantly improves 3D grounding accuracy over single‑view RGB‑D, especially under occlusion.
- The modular design (VLM planning + motion primitives) avoids costly data collection and embodiment‑gap issues.

### Strengths
- **Zero‑shot capability**: No task‑specific training or demonstrations needed.
- **Robust 3D grounding**: Multi‑view triangulation + ray voting reduces depth ambiguity and occlusion impact.
- **Generalizable to tool‑use**: Atomic action library aligns to novel scenes.
- **Closed‑loop failure recovery**: Verification and replanning enable robust long‑horizon execution.
- **Comprehensive real‑world evaluation**: Multiple tasks, baselines, and ablations.

### Limitations
- **VLM reliance**: Errors in 2D semantic grounding (task decomposition, affordance selection) propagate to execution failures.
- **Motion planner fragility**: Kinematic singularities, collision‑checker timeouts, or unstable grasps can cause failures and latency.
- **No in‑hand manipulation**: The system does not handle actions like rotating an object inside the hand or using scissors.
- **Limited scalability**: Offline atomic action library covers only pre‑defined skill categories; real‑time generation is not explored.
- **Small sample sizes**: Some task evaluations have only 3–6 trials, limiting reliability of reported success rates.
