## Abstract
Modern Vision-Language-Action (VLA) models often fail to generalize to novel setups, such as altered camera viewpoints or robot morphologies, because they are typically conditioned only on current observations and language instructions. By ignoring the underlying system configuration as a variable, these models implicitly assume a fixed execution context encountered during training, necessitating data-intensive fine-tuning for any new environment. In this work, we introduce In-Context World Modeling (ICWM), a framework that treats system identification as an in-context adaptation problem. ICWM enables robot policies to autonomously infer essential system variables from a short history of self-generated, task-agnostic interactions. Unlike traditional In-Context Learning that uses demonstrations to specify what task to perform, ICWM leverages the context window to understand how the system operates. By processing these interactions before task execution, the model implicitly captures the world dynamics of the current system, enabling adaptation to novel configurations without parameter updates. Extensive experiments in simulation and on real-world robot platforms demonstrate that ICWM significantly outperforms standard VLA baselines on novel camera viewpoints.

- **Motivation**: Inspired by humans who calibrate to unknown controls through a short exploratory phase, the authors propose **In-Context World Modeling (ICWM)**, which repurposes the context window for system identification rather than behavior specification. The robot performs a brief, task‑agnostic self‑probing phase; the resulting observation–action–next‑observation triplets are prepended as context, allowing the model to infer the current system dynamics **without any parameter updates or task‑specific demonstrations**.

## 2. Methodology

- **Core Idea**: Transform the VLA policy from a static mapping \( \pi_\theta(a_t \mid o_t, l) \) into an adaptive inference engine that explicitly conditions on the latent configuration ψ via in‑context interaction clips T.
- **Formalization**:  
  \[
  a_t \sim \pi_\theta(a_t \mid \Psi(T), o_t, l)
  \]
  where \( \Psi(T) \) denotes hidden states induced by the interaction context \( T = \{ (o_s^i, a^i, o_e^i) \}_{i=1}^N \).  
  The shared parameters \( \theta \) learn to extract dynamics information from T during training.
- **Training**:  
  - Data is collected across **diverse system configurations** (e.g., 8 azimuthal camera angles).  
  - Each training sample is constructed by prepending N = 5 task‑agnostic interaction clips (randomly sampled from the same episode) to the task query (current observation + instruction).  
  - Loss: standard next‑action prediction (negative log‑likelihood).  
  - The model must learn to **implicitly model the action‑to‑observation mapping** from the context to predict task actions correctly.
- **Test‑Time Inference** (two phases):  
  1. **Active Probing**: The robot executes 20 random, task‑agnostic movements (sampling target poses in the safe workspace) and records the resulting transitions.  
  2. **In‑Context Execution**: The N = 5 triplets are prepended as context; the Transformer attends to them to build configuration‑aware hidden states, then processes the task query to generate actions.

## 6. Main Conclusions and Findings

- ICWM **significantly outperforms** all baselines on OOD viewpoints:  
  - Simulation: +13.0% over MV (average), +9.5% over EXP.  
  - Real‑world: +71% over MV (average success rate 88% vs 17% under viewpoint shift).  
- Implicit world modeling from interaction context is **more effective** than providing ground‑truth camera angles explicitly.  
- The benefit **increases with task complexity** (largest gain on LIBERO‑Long: +26.3% OOD).  
- False context (viewpoint mismatch) **degrades** performance (worse than no context), confirming genuine model adaptation.  
- Probing strategy matters less than having any context; all four strategies improve over MV baseline by 15–27%.  
- ICWM generalizes to semantic scene variations and robot morphological changes (e.g., +60% over MV with 80 mm spacers; on WindowX margin grows to 34 points at larger kinematic offset).

## 7. Strengths

- **Novel formulation**: Reframes VLA generalization as a test‑time system identification problem, a previously underexplored failure mode.  
- **Practical**: No parameter updates, no task‑specific demonstrations, minimal extra hardware/calibration.  
- **Efficient**: Shares parameters with the VLA backbone; probing overhead is low (~5 s per deployment) and can be amortized.  
- **Robust validation**: Simulation + real‑world, extensive ablations, and generalization beyond camera shifts to morphology and semantics.  
- **Interpretability**: t‑SNE analysis shows learned representations are identifiable and structured by configuration.

## 8. Limitations

- **Viewpoint‑specific failures**: Some OOD viewpoints (e.g., 135°) remain challenging for all methods due to occlusion/field‑of‑view issues.  
- **Probing time**: 5–6 seconds of pre‑execution probing may be unacceptable in time‑critical applications.  
- **Semantic generalization**: Gains over MV are more moderate for texture/distractor changes, possibly due to limited diversity in training data.  
- **Safety of probing**: Random movements require a predefined safe workspace; in unstructured environments, self‑probing could cause collisions.  
- **Dynamic environments**: Not tested in scenarios where the configuration changes mid‑episode (e.g., moving camera, variable payload).  
- **Human interaction**: No evaluation in shared human‑robot workspaces or with real‑time human intervention.
