evidence: Uses generative world models to generate dexterous human hand manipulation data for VLA models
tldr: "Scaling dexterous manipulation requires generalization across objects, scenes, and tasks, but existing data sources trade off between scale and alignment. Wh0 uses generative video world models to produce a 50k-episode dataset of egocentric human-hand manipulation videos, converted into robot supervision via hand motion reconstruction and visual editing. Co-trained with limited real robot data, Wh0 improves zero-shot success on unseen tasks from 8.3% to 38.9%. Scalable generation and scene/embodiment alignment are key to performance gains."

## Abstract
Scaling dexterous manipulation requires generalization across objects, scenes, and tasks, yet existing data sources face a trade-off between scale and scene/embodiment alignment: teleoperation data is well aligned with robot deployment but expensive to collect; simulation is scalable but limited by the sim-to-real gap; and real egocentric videos scale effectively but remain misaligned with robot deployment. We propose Wh0, a framework that uses generative video world models as scalable and controllable sources of egocentric human-hand manipulation data to unlock the manipulation capabilities of pretrained dexterous VLA models. Conditioned on language, objects, and scenes, Wh0 uses a generative world model to produce WM-H, a 50k-episode dataset of egocentric human-object interaction videos. Wh0 then converts the generated videos into robot-trainable supervision through hand motion reconstruction and visual editing. Co-trained with a limited amount of real robot data, WM-H adapts pretrained VLA models to dexterous manipulation deployment. Across 18 real-world dexterous manipulation tasks, compared with a model post-trained only on robot data, Wh0 improves zero-shot success on unseen tasks from 8.3% to 38.9%. Ablation studies further show that scalable generation and scene/embodiment alignment are key drivers of performance gains. Videos and open-source code can be found on our project website: https://chenyt31.github.io/wh0.github.io/.

# Wh0: Generative World Models as Scalable Sources of Egocentric Human Hand Manipulation Data

## Core Problem and Overall Meaning

- **Motivation:** Scaling dexterous manipulation requires generalization across objects, scenes, and tasks. Existing data sources face a trade-off: teleoperation is aligned but expensive; simulation scales but suffers sim-to-real gap; real egocentric videos scale but are misaligned with robot deployment (scene and embodiment gaps).
- **Overall meaning:** Wh0 proposes using generative video world models as a controllable, scalable source of egocentric human-hand manipulation videos. These videos are converted into robot-suitable supervision via hand motion reconstruction and visual editing, enabling pretrained VLA models to generalize to unseen dexterous tasks with minimal real robot data.

## Methodology

- **Core idea:** Use a generative video world model (Wan-I2V-A14B) to synthesize egocentric human-object interaction videos conditioned on language, objects, and scenes. Extract 3D hand poses as action labels. Edit selected frames to replace human hands with realistic robot hands for embodiment alignment. Co-train the generated dataset (WM-H) with a small set of teleoperated robot demonstrations to adapt a human-video-pretrained VLA policy.
- **Key technical details:**
  - **WM-H dataset construction (50k episodes):**
    1. **Instruction generation:** Dual-agent LLM system discovers nouns/adjectives and uses balanced sampling to compose commands (e.g., "pick the red mug") with high diversity (noun h-index 201, adj h-index 117).
    2. **Scene-aligned image editing:** Capture background in robot workspace (same viewpoint/resolution). Use Qwen-Image-Edit to insert specified objects, preserving scene layout and scale.
    3. **Image-to-video generation:** Use Wan-I2V-A14B plus LightX2V LoRA (4 inference steps) to animate the edited image. Append a Qwen3-VL-generated description of expected hand-object dynamics to the prompt.
    4. **Robot-hand editing:** Replace human hand with a realistic robot hand (white back shell, black palm, black fingertips, silver forearm) on selected frames using Qwen-Image-Edit, preserving pose, motion, and object interaction.
    5. **Motion extraction:** Use HaWoR to reconstruct 3D MANO hand parameters (wrist pose, joint rotations) from each human-hand video.
  - **Policy architecture (VITRA-style):**
    - Vision-language backbone: PaliGemma2-3B encodes observation and instruction. A learnable cognition token is extracted as conditioning feature.
    - Action decoder: Diffusion-based DiT-B network predicts future hand motions (16 steps) in unified MANO space.
    - Action space (right hand): relative wrist translation (ΔT), rotation (ΔR), and 15-DoF joint rotations (θ_h) → dimension 102.
    - Training loss: noise-prediction MSE (Eq. 2).
  - **Training strategy:**
    - Initialize from VITRA (pretrained on Ego4D, Epic-Kitchens, EgoExo4D, Something-Something-V2).
    - Co-finetune on a mixture: 28% teleop (400 demos), 68% WM-H, 4% embodiment-aligned WM-H (WM-H EA). Freeze vision encoder; update backbone and action decoder. Learning rate 1e-5, weight decay 0.1, 40k steps on 4 H200 GPUs.

## Experiments

- **Datasets/scenarios:** Real-world evaluation on Unitree G1 humanoid with Inspire dexterous hands. 18 tasks (e.g., grasp, place, wipe, tool use) across four backgrounds (one seen, three unseen). 20 trials per task, randomized object poses. Zero-shot evaluation (no task-specific demonstrations).
- **Baselines:**
  - π0.5 (robot-data pretrained + teleop)
  - VITRA (human-video pretrained + teleop)
  - VITRA Real Version (teleop + real egocentric HOI4D videos)
- **Key results:**
  - Wh0 achieves 38.9% success rate vs VITRA 8.3% (4.7× improvement) and VITRA Real Version 21.4%.
  - Ablations show scene alignment, embodiment alignment, and larger scale (50k > 25k > 5k) all improve task success and hand-object grounding.
  - Analysis: WM-H unlocks human-video pretraining priors; without pretraining, near-zero success.

- **Number of runs:** 18 tasks × 20 trials each per method. Standard deviations reported.
- **Ablations:** Scene alignment removal, embodiment alignment removal, data scale (5k/25k/50k), pretraining source, teleop-only, WM-H only, etc. Also user study (72 participants) evaluating video quality and hand editing fidelity. Failure case analysis provided.
- **Coverage:** Sufficient and fair. Includes strong baselines (π0.5, VITRA) and real-egocentric video comparison. Ablations isolate key components. However, limited to single-arm tasks.

## Main Conclusions and Findings

1. Wh0 significantly improves zero-shot dexterous manipulation (8.3% → 38.9%) by using world-model-generated human-hand videos as post-training data.
2. Scene alignment (same viewpoint/workspace), embodiment alignment (robot hand editing), and data scale are critical for performance.
3. WM-H primarily unlocks manipulation priors from human-video pretraining, not learning dexterous skills from scratch.
4. Generative world models can serve as scalable, deployment-aligned data sources for VLA policies.

## Strengths

- **Scalability:** Data generation depends on compute, not human labor (5.44 GPU-hours per 1k videos).
- **Alignment techniques:** Scene and embodiment alignment reduce gaps between synthetic data and real robot deployment.
- **Empirical rigor:** Extensive real-world evaluation with ablations; user study validates video quality.
- **Open-source:** Code and videos released.

## Limitations

- **Video generation quality:** Physical implausibility, unexpected objects, inconsistent long-horizon motion.
- **Hand reconstruction accuracy:** Occlusions degrade finger pose quality, introducing noisy supervision.
- **Human-robot morphology mismatch:** Robot hand larger; may disturb objects unintentionally.
- **Dependence on strong pretraining:** WM-H offers little benefit without human-video-pretrained backbone.
- **Task scope:** Single-arm pick-and-place only; bimanual, tool-use, longer-horizon tasks not explored.
- **Bias risks:** Generation may overrepresent certain objects/scenes; failure cases not fully quantified in training impact.
