## Abstract
Long-horizon robot manipulation remains challenging because similar observations may occur at different execution stages, while the appropriate action depends on previously completed operations. Memory can address this ambiguity by enabling policies to infer task progress from execution history. However, existing memory-augmented approaches often either retain dense histories that require compression or rely primarily on recent context that may discard earlier task-relevant events. In this work, we propose propose KEMO, a lightweight plug-in memory framework that automatically selectively preserves keyframes associated with task-relevant state changes for VLA policies. KEMO combines robot kinematics with visual filtering to detect events, encodes the selected keyframes as compact temporally ordered memory tokens, and integrates them with current visual features through cross-attention and gated residual fusion for VLA training. The detected events also define higher-weight training samples near critical transitions. We evaluate KEMO on various real-world dual-arm manipulation tasks spanning 2 to 6 scored subtasks, and trajectory length ranging from 830 steps to 2846 execution steps (durations from 28 to 95 seconds). Compared with the memory-free baseline (e.g., $π_{0.5}$), KEMO improves aggregate Task Success Rate by 23.6\% and Stage Completion Rate by 34.1\%. Ablations show that event-driven keyframe selection outperforms uniform sampling and recent-frame retention, while the proposed gated fusion and keyframe-aligned loss weighting provide complementary gains.

## Detailed Summary (auto-generated)

**1. Core Problem and Overall Meaning**  
Long-horizon robot manipulation suffers from *stage ambiguity*: visually similar observations can appear at different task stages, requiring different actions depending on what has already been completed. Using only current observations, VLA policies cannot disambiguate these states. Memory of past execution can resolve ambiguity, but existing approaches are either dense (compressing redundant history) or rely on heavy online VLM reasoning. KEMO is a lightweight, plug-in memory framework that selectively preserves only *event keyframes*—moments of task-relevant state change—and injects them into the VLA policy to provide explicit historical evidence.

**2. Methodology**  
- **Event-Driven Keyframe Detection**  
  - *Kinematic saliency score*:  
    \( S_t = \frac{1}{1 + \bar{\delta}_t} \), where \(\bar{\delta}_t\) is the mean joint displacement over a sliding window of length \(w\). High scores occur when the robot slows down, typical at manipulation events (grasping, placing).  
  - *Causal peak detection*: Local peaks in \(S_t\) become candidate keyframes.  
  - *Visual deduplication filter*: Compares each candidate image with the last accepted keyframe using frozen DINOv2 embeddings. A candidate is accepted only if cosine dissimilarity \(v_i > \epsilon\) (threshold).  
  - A refractory period \(r\) prevents multiple detections within the same transition.  

- **Keyframe Memory Integration**  
  - *Encoding*: Each accepted keyframe image is encoded via SigLIP, then spatially pooled (4×4 → 16 tokens per keyframe).  
  - *Memory bank*: Maintains the \(K\) most recent keyframes; missing slots are padded with a binary mask.  
  - *Temporal positional encoding*: Learnable embeddings per bank slot indicate stage order.  
  - *Cross-attention*: Current frame tokens (queries) attend to concatenated keyframe tokens (keys/values). Output is fused via a learned residual gate: \(\hat{X}_t = X_t + \sigma(g([X_t; X'_t])) \cdot X'_t\), with negative bias initialization to suppress memory early in training.  
  - The updated visual tokens pass unchanged to the language model.  

- **Keyframe-Consistent Training**  
  - The *detected keyframes* are reused to define a loss-weighting scheme: \( w_t = \lambda \) if \( \min_i |t - k_i| \le \delta \), else 1.  
  - The per-step flow-matching loss \(\ell_t\) is multiplied by \(w_t\), emphasizing critical transitions where errors are most consequential.

**6. Main Conclusions and Findings**  
- KEMO improves aggregate TSR by **23.6%** (27.8% → 51.4%) and SCR by **34.1%** (42.3% → 76.4%) over \(π_{0.5}\).

**7. Strengths**  
- **Lightweight and plug-in**: adds only keyframe detection and memory integration; does not require modifying the VLA backbone.  
- **Task-agnostic detection**: uses kinematic and visual cues, no need for predefined subtask labels or per-task tuning of semantic detectors.  
- **No additional online VLM inference** (avoids latency and hallucination).  
- **Real-world validation**: diverse, long-horizon tasks with up to 6 subtasks and 95-second trajectories.  
- **Clear ablations** that isolate the contribution of each design choice.

**8. Limitations**  
- **Hyperparameter sensitivity**: sliding window \(w\), peak-detection window, refractory period \(r\), and memory bank capacity \(K\) require manual tuning per task.  
- **Fixed \(K\)**: currently set by expected number of stage transitions; does not adapt to trajectory length or variation.  
- **No learned event selection** – the detector uses handcrafted rules; future work could incorporate learned importance scores.  
- **Limited to single-task fine-tuning**: experiments are per-task; cross-task generalization not evaluated.  
- **Potential bias** from demonstration quality and limited random initializations (12 trials per task).
